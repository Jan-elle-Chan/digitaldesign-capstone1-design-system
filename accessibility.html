Accessibility

Web Accessibility

Websites must be designed so that persons with the following permanent or temporary conditions may access them and have a seamless experience:

  • Blindness or poor vision
  • Deafness or hearing difficulties
  • Cognitive and learning disabilites
  • Technological disdavantage such as poor Internet or low memory
  • Environment that is not conducive to accessing web content

Page Element Guidelines for accessibility HTML/ CSS required
Images
  • Use alt tags for images that describe image if it fails to download or cannot be found
  • Use caption tag to provide image details
 <figure> <img src="red_box.jpg" alt="Red box"> <figcaption>Fig.1 - A Red box</figcaption> </figure>
Tables Always use caption tag <table> <caption>This is a table</caption>
Iframes Always use a clear title <iframe title="Code for cards">
Forms Dealt with in detail in Forms section Navigation guidelines
Audio and Video
  • Allow usrs to choose their own pace, transcripts and playback controls
  • Provide options for captions and volume
Navigation Dealt with in detail in Navigation section Navigation guidelines
Text
  • Use simple, clear language that is easy to understand
  • Group content and place under headings so users can skim and select
 <H1> to <H6> tags
